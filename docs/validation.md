## Interview Structure

We conducted user interviews using a structured set of questions designed to identify pain points and validate SmartSplit’s features.

**Questions used:**

1. Have you ever shared expenses with roommates or friends? How did you manage it?
2. What is the most annoying or stressful part of splitting expenses with other people?
3. Have you used any apps (like Splitwise)? What did you like or dislike about them?
4. If you had to track shared expenses right now, how would you do it?
5. SmartSplit automatically splits expenses and updates balances in real time. Would that be useful to you? Why or why not?
6. The app allows equal splits and custom percentages. Would you use custom splits or keep it simple?
7. The app can automatically add recurring expenses. How important is this feature to you?
8. SmartSplit simplifies debts to minimize transactions. Do you understand this idea, and would you trust it?
9. You can select participants and mark payments as completed manually. Does this match how you would use the app?
10. What is the ONE feature that would make you definitely use this app, and what could make you stop using it?

## User Interviews

We conducted **2 user interviews** with individuals who have experience sharing expenses with friends.

---

### Interview 1

**Background:**
User has experience sharing expenses and managing them manually.

**Responses:**

1. Shared expenses before; found it challenging due to awkward reminders.
2. Main issue is having to remind others to pay.
3. Has not used similar apps.
4. Uses calculator + group chat.
5. Finds automatic splitting and reminders very useful.
6. Would use both equal and custom splits depending on the situation.
7. Considers recurring expenses very important for monthly tracking.
8. Understands and values debt simplification.
9. Suggested adding payment confirmation (e.g., screenshot proof).
10. Identified payment confirmation as a key feature.

---

### Interview 2

**Background:**
User has experience splitting expenses, typically equally.

**Responses:**

1. Has shared expenses; usually splits evenly.
2. Main issue is people forgetting to pay.
3. Has not used similar apps.
4. Splits manually and communicates via messages.
5. Finds automation and reminders useful.
6. Prefers equal split as default but supports custom options.
7. Does not consider recurring expenses essential, but sees some value.
8. Understands and agrees with debt simplification.
9. Finds the approach consistent with expected usage.
10. Would only use the app if it is simple, fast, and adds clear value.

---

## Key Insights

From the interviews, several patterns emerged:

* The main pain point is **having to remind others to pay**, creating social discomfort.
* Current methods are **manual and inefficient** (calculator + messaging).
* Users strongly value **automation, reminders, and real-time updates**.
* **Equal split should be the default**, with optional customization.
* **Trust in payment confirmation** is important and may require additional features.
* The app must be **simple and faster than existing methods** to ensure adoption.

---

## Impact on Product Backlog

Based on the feedback, the Product Backlog was refined as follows:

### Reordered PBIs

* Increased priority of:

  * Notifications and reminders
  * Real-time balance tracking
  * Simple and fast expense creation

### New PBIs Added

* **Payment confirmation system**
  "As a user, I want to confirm payments (e.g., via screenshot) so that all participants trust that debts are settled."

* **Default equal split with optional customization**
  "As a user, I want equal split to be the default so that adding expenses is fast and intuitive."

* **UX simplification improvements**
  "As a user, I want a fast and intuitive interface so that using the app is easier than manual methods."

### Adjusted Priorities

* Recurring expenses were slightly deprioritized, as they are not essential for all users.

---

## Conclusion

The interviews validated SmartSplit’s core objective of reducing friction in shared expense management.

The results emphasized the importance of automation, simplicity, and trust, which were incorporated into the refinement of the Product Backlog to better align the product with user needs.
